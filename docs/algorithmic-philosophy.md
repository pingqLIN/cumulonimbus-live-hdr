# High Anvil Accretion

High Anvil Accretion treats a cumulonimbus tower as a living density field rather than an illustrated object. The cloud is generated by layered cellular noise, slow domain warping, altitude masks, and erosion thresholds that change over time. The edge is the main subject: it grows and disappears by fractions, with motion so slow that the viewer feels pressure changing rather than animation playing.

The system is intentionally self-iterative. Each frame converges a persistent density buffer toward seeded atmospheric equations, while slow wind shear advects the previous state before condensation or evaporation is applied. Billows are not sprites; they emerge from thresholded turbulence, vertical lift, anvil spread, and memory. The result should feel carefully engineered, as if the algorithm has been tuned through many passes to avoid noisy randomness and preserve meteorological gravity.

HDR is treated as a luminance language. The silver lining and sunlit upper rim are allowed to exceed ordinary SDR values before export, while the browser preview tonemaps them for ordinary displays. The final stream path should keep those highlights as high-value information, not merely brightened SDR.

The current prototype favors a single portrait segment matching the reference video's vertical framing, with both a persistent 2D HDR field and a browser-backed 3D bubble preview. Later iterations can add WebGPU acceleration, OBS/NDI/Spout output, or segmented live-loop rendering once the movement vocabulary is right.
